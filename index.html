Introducción

HTML (HyperText Markup Language) es el lenguaje de marcado estándar para la creación de páginas web. Define la estructura y el significado del contenido en una página web, permitiendo a los navegadores web interpretar y mostrar el contenido de manera adecuada.

Esta documentación proporciona una visión general de los conceptos clave y prácticas recomendadas en HTML, desde la estructura básica hasta la accesibilidad y semántica, para ayudar a los desarrolladores a crear contenido web bien estructurado y fácil de entender tanto para los navegadores como para los usuarios.

Estructura Básica de un Documento HTML

La estructura básica de un documento HTML consta de elementos que definen la información y la presentación de la página web.


  <!DOCTYPE html>
   <html>
     <head>
      <title>Título de la Página</title>
     </head>
     <body>
      <!-- Contenido de la página -->
     </body>
   </html>

A continuación, se desglosa el ejemplo:

  • <!DOCTYPE html>: Esta declaración de tipo de documento (doctype) es la primera línea de cualquier documento HTML5 y le indica al navegador que el documento debe interpretarse como HTML5.
  • <html>: Este es el elemento raíz de una página web y contiene todo el contenido visible y oculto de la página.
  • <head>: El elemento <head> contiene metainformación sobre el documento, como el título de la página, enlaces a hojas de estilo (CSS), scripts de JavaScript, y otras configuraciones que no se muestran directamente en la página.
  • <title>: El elemento <title> define el título de la página web, que se muestra en la barra de título del navegador y en los resultados de búsqueda.
  • <body>: El elemento <body> contiene todo el contenido principal de la página web que se muestra al usuario, como texto, imágenes, enlaces, formularios, etc. Esta estructura básica es la base sobre la cual se construyen todas las páginas web y es crucial para asegurar que el contenido se presente de manera consistente y accesible en diferentes navegadores y dispositivos.
Elementos Semánticos

Los elementos semánticos son aquellos que tienen un significado inherente y ayudan a describir el tipo de contenido que contienen. Estos elementos no solo mejoran la accesibilidad y la experiencia del usuario, sino que también facilitan la interpretación de la página por parte de los motores de búsqueda y las tecnologías de asistencia, como los lectores de pantalla.

En el ejemplo:


  <article>
      <h1>Titular del Artículo</h1>
      <p>Contenido del artículo...</p>
  </article>

Se utiliza el elemento <article>, que es un elemento semántico que representa un componente independiente de contenido que tiene sentido por sí mismo, como un post de blog, una noticia o un comentario. En este caso, el <article> contiene un título (<h1>) y un párrafo (<p>), que juntos conforman el contenido completo del artículo.

Los elementos semánticos como <article>, <section>, <nav>, <aside>, <header>, <footer>, entre otros, proporcionan una estructura clara y significativa al contenido, lo que facilita la navegación y la comprensión tanto para los humanos como para las máquinas. Además, estos elementos pueden ser estilizados y manipulados de manera más precisa con CSS y JavaScript, lo que permite un mayor control sobre la presentación y funcionalidad de la página web.

Listas

Las listas en HTML pueden ser ordenadas (<ol>) o desordenadas (<ul>), y cada elemento de la lista se envuelve en un <li>.
Ejemplo:


  <ul>
    <li>Elemento 1</li>
    <li>Elemento 2</li>
    <li>Elemento 3</li>>
  </ul>
Enlaces e Imágenes

Dos elementos fundamentales en HTML que permiten la interacción y la presentación de contenido multimedia en una página web.

Los enlaces (<a>) son elementos que permiten la navegación entre páginas web o a diferentes secciones de la misma página. Se definen con el atributo href, que especifica la URL a la que apunta el enlace. El texto o contenido dentro del elemento <a> se convierte en un hipervínculo interactivo que los usuarios pueden hacer clic para acceder al destino.

Las imágenes (<img>) son elementos incrustados que permiten mostrar gráficos y fotos en una página web. Se definen con el atributo src, que especifica la ruta o URL de la imagen que se va a mostrar. También es importante incluir el atributo alt, que proporciona una descripción textual de la imagen para casos en los que la imagen no se puede cargar o para lectores de pantalla que ayudan a las personas con discapacidades visuales.

En el ejemplo:


  <a href="https://www.ejemplo.com">Visita Ejemplo.com</a>
  <img src="imagen.jpg" alt="Descripción de la imagen">

Se muestra cómo se crea un enlace a otra página web y cómo se inserta una imagen en la página actual. El enlace lleva al usuario a "https://www.ejemplo.com" cuando hacen clic en "Visita Ejemplo.com". En el caso de la imagen se carga desde el archivo "imagen.jpg" y se describe como "Descripción de la imagen" para los lectores de pantalla y en caso de que la imagen no se pueda cargar.

Estos elementos son esenciales para la creación de páginas web interactivas y accesibles, y su uso correcto mejora la experiencia del usuario y la optimización de motores de búsqueda (SEO).

Formularios

Los formularios (<form>) permiten recoger datos del usuario a través de campos de entrada (<input>, <textarea>, etc.) y botones de acción (<button>, <submit>).

Permiten recopilar información del usuario a través de una interfaz web. Los formularios son una parte integral de muchas aplicaciones web y sitios web, ya que proporcionan una forma de enviar datos a un servidor para su procesamiento, almacenamiento o visualización.

Aquí hay cinco buenas prácticas al estructurar un formulario en HTML:

  • Uso de Etiquetas (<label>): Proporcionar etiquetas claras y descriptivas para cada campo de entrada para que los usuarios comprendan qué información se espera que ingresen.
  • Validación del Lado del Cliente: Utilizar atributos como pattern y required para validar la entrada del usuario antes de enviar el formulario, reduciendo así las solicitudes innecesarias al servidor.
  • Accesibilidad: Asegurarse de que todos los elementos del formulario sean accesibles, utilizando atributos como aria-* para mejorar la navegación con tecnologías de asistencia.
  • Grupos de Botones: Agrupar botones relacionados con <fieldset> y <legend> para mejorar la organización y la accesibilidad.
  • Uso de Placeholders: Utilizar placeholders (placeholder attribute) para dar pistas sobre el formato esperado de la entrada, pero evitar confiar únicamente en ellos para la validación.

En el ejemplo:


  <form action="/ruta-al-servidor" method="post">
    <label for="nombre">Nombre:</label>
    <input type="text" id="nombre" name="nombre">
    <input type="submit" value="Enviar">
  </form>

Se muestra un formulario simple con un campo de entrada para el nombre del usuario y un botón de envío. El atributo action especifica la URL a la que se enviarán los datos del formulario cuando se haga clic en el botón de envío, y el atributo method indica que los datos se enviarán mediante el método HTTP POST.

El elemento <label> está asociado con el campo de entrada mediante el atributo for, que coincide con el valor del atributo id del campo de entrada. Esto mejora la accesibilidad al permitir que los usuarios seleccionen el campo de entrada haciendo clic en la etiqueta. El botón de envío (<input type="submit">) es el elemento que permite al usuario enviar el formulario.

Siguiendo estas buenas prácticas, se puede crear un formulario HTML que sea fácil de usar, accesible y eficiente en términos de rendimiento y experiencia del usuario.

Tablas

Las tablas (<table>) se utilizan para presentar datos en filas y columnas, con elementos como <tr> para filas, <th> para encabezados y <td> para celdas de datos.

En este ejemplo:


          

    

        Encabezado 1
        Encabezado 2
    

    

        Dato 1
        Dato 2

Se realizó con el siguiente código


  <table>
    <tr>
        <th>Encabezado 1
        <th>Encabezado 2
    </tr>
    <tr>
        <td>Dato 1
        <td>Dato 2
    </tr>
  </table>
Accesibilidad y Semántica

Destaca la importancia de utilizar elementos HTML de manera semántica y de considerar la accesibilidad al diseñar y estructurar una página web. La semántica se refiere a la elección de elementos HTML que transmiten el significado del contenido, mientras que la accesibilidad se centra en garantizar que la página web sea utilizable por todas las personas, independientemente de sus habilidades físicas o sensoriales.

En el ejemplo:


  <header>
    <h1>Logo de la Empresa</h1>
    <nav>
        <ul>
            <li><a href="#inicio">Inicio</a></li>
            <li><a href="#acerca">Acerca de</a></li>
        </ul>
    </nav>
  </header>

Se observan varias prácticas de semántica y accesibilidad:

  • Elemento <header>: Este elemento semántico se utiliza para contener el contenido introductorio o de navegación principal de una sección o página. En este caso, el <header> contiene el logo de la empresa y la navegación principal.
  • Elemento <h1>: Representa el título principal de la página o sección. En este ejemplo, el <h1> se utiliza para el logo de la empresa, que es un título importante que identifica la marca.
  • Elemento <nav>: Este elemento semántico se utiliza para agrupar los principales enlaces de navegación de la página. En este caso, contiene una lista desordenada (<ul>) con enlaces a diferentes secciones de la página.
  • Elemento <ul> y <li>: La lista desordenada (<ul>) se utiliza para agrupar los elementos de la lista (<li>), que en este caso son los enlaces de navegación.
  • Elemento <a>: Los enlaces (<a>) permiten a los usuarios navegar a diferentes partes de la página o a otras páginas web. Los atributos href especifican la ubicación a la que apuntan los enlaces.

Al utilizar estos elementos semánticamente, se mejora la accesibilidad al permitir que las tecnologías de asistencia, como los lectores de pantalla, interpreten la estructura de la página y presenten la información de manera coherente. Además, los motores de búsqueda pueden entender mejor el contenido y su relevancia, lo que puede afectar el ranking de la página en los resultados de búsqueda.

En resumen, el ejemplo demuestra cómo la semántica y la accesibilidad pueden trabajar juntas para crear una página web que sea fácil de usar y entender para todos los usuarios, y que sea amigable con los motores de búsqueda.
